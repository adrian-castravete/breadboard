Breadboard\new (dontStart)
Constructor. Create the main Breadboard object, a simple engine on top of Löve2D

Parameters:

  • dontStart boolean whether to create the cart and start running
Breadboard\draw (srcX, srcY, spX, spY, offX, offY, rotationAngle, scale, a, col)
Draw (part of) the map to screen. Given some source coordinates along with a width and height, draw as many tiles to the screen to a specified coordinate.

Parameters:

  • srcX number the start coordinate to draw the map from (X coordinate)
  • srcY number the start coordinate to draw the map from (Y coordinate)
  • spX number the amount of tiles to draw to the screen (X coordinate)
  • spY number the amount of tiles to draw to the screen (Y coordinate)
  • offX number the position on screen to draw to (X coordinate)
  • offY number the position on screen to draw to (Y coordinate)
  • rotationAngle number rotate the map while drawing it to the screen (default 0)
  • scale number the size of the map while drawing to the screen (default 1)
  • a number the opacity of the thing to draw (default 1)
  • col table the colour effect (default is white {1, 1, 1})
Breadboard\tile (tsetID, srcX, srcY, dstX, dstY, spX, spY, a, col, offX, offY, fX, fY)
Draw new tiles from a tileset into the map data. Given a tileset copy one or more tiles to the map data.

Parameters:

  • tsetID string the Identification tag for the loaded tileset
  • srcX number the start coordinate to draw the from the tileset (X coordinate)
  • srcY number the start coordinate to draw the from the tileset (Y coordinate)
  • dstX number the start coordinate to draw on the map (X coordinate)
  • dstY number the start coordinate to draw on the map (Y coordinate)
  • spX number the amount of tiles to draw to the tile map (X coordinate) (default 1)
  • spY number the amount of tiles to draw to the tile map (Y coordinate) (default 1)
  • a number the opacity of the thing to draw (default 1)
  • col table the colour effect (default is white {1, 1, 1})
  • offX number finer control over where to draw on the map (X coordinate) (default 0)
  • offY number finer control over where to draw on the map (Y coordinate) (default 0)
  • fX boolean whether to flip horizontally (default false)
  • fY boolean whether to flip vertically (default false)
Breadboard\tileClear (dstX, dstY, spX, spY, a, col)
Clear one or more tiles from the tile map.

Parameters:

  • dstX number the start coordinate to clear on the map (X coordinate)
  • dstY number the start coordinate to clear on the map (Y coordinate)
  • spX number the amount of tiles to draw to the tile map (X coordinate) (default 1)
  • spY number the amount of tiles to draw to the tile map (Y coordinate) (default 1)
  • a number the opacity of the thing to draw (default 0)
  • col table the colour effect (default is black {0, 0, 0})
Breadboard\clearScreen (colours, red, green, blue, alpha)
Clear the screen. It's advisable to always have this in the start of the tick function. Whatever is given to this function is passed to love.graphics.clear. So you can pass either one table or up to four numbers. The default is colour black.

Parameters:

  • colours table a table containing the red, green, blue values and an optional alpha on their respective positions
  • red number value for the red channel
  • green number value for the green channel
  • blue number value for the blue channel
  • alpha number value for the alpha channel
Breadboard\makeTileset (fileName)
Make a tileset from a file. Load a file pointed to by fileName and check whether it has already been loaded. In case it has, return the existing tileset, otherwise create a tileset from the file, cache it and return it.

Parameters:

  • fileName string the given file name

Returns:

    string tileset ID

Breadboard\printXY (x, y, text, col)
Print text on screen. Print some text on screen using the default font. This is very basic and should only be used for debugging reasons, or not at all.

Parameters:

  • x number the X coordinate of where to start writing
  • y number the Y coordinate of where to start writing
  • text string the text to write
  • col table the colour to use (default white {1, 1, 1})
Breadboard\tileDump (fileName)
Write the tilemap to disk. Given a file name write the current state of the tile map to disk. This is pretty heavy, so it should be avoided for every frame. It's mainly for debugging reasons.

Parameters:

  • fileName string the file name to write to. (PNG)
Breadboard\config (options)
Change configuration stuff. Some changes may only be changed before the start of the engine, others can be change whilst it is running.

Parameters:

  • options table

    the configuration table; the known confiuration entries are:

    • disableTouch - disable the gamepad that appears on android; Good for when you use Löve2D's touch capabilities. (default false)
    • showMouse - by default the mouse is hidden, set this to true to leave the mouse cursor visible
Breadboard\viewport ()
Return some viewport information. Return some crucial?! information about the viewport.

Returns:

    table

    a table containing the following values:

    • offsetX and offsetY - Given that the viewport is dynamic this shows where exactly on screen the first pixel of the internal viewport square
    • width and height - The viewport dimensions
    • zoom - The current scale/zoom level
Breadboard\animate (delay, fnProgress, fnDone)
Start an animation coroutine. Given a delay, a progress function and a done function, until the delay ends, execute the progress function (passing it the elapsed percentage) and finally execute the done function.

Parameters:

  • delay number the amount in milliseconds to execute this animation for.
  • fnProgress function the progress function
  • fnDone function the done function
Breadboard\lerp (a, b, ratio)
Linear interpolation function. Interpolate two numbers at a specific percentage.

Parameters:

  • b number the second value
  • ratio number the ratio between the two as a value between 0 and 1

Set or Get the screen size. If a value v is given, set the internal viewport size to v x v.
